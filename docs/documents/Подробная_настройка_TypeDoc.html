    Руководство по настройке TypeDoc

    Назначение: пошаговая инструкция для адаптации преднастроенного TypeDoc под ваш проект. Конфигурация уже работает; требуется обновить только отдельные параметры.

    Предварительные условия

    В проекте должны присутствовать:

    • typedoc.json — конфигурация TypeDoc (обновлённая, см. ниже).
    • tsconfig.build.json — сборочная конфигурация TypeScript с declaration: true и declarationMap: true.
    • src/index.ts — вход публичного API.
    • README.md — главная страница документации.
    • Каталог manuals/ — для пользовательских руководств в формате Markdown (если используются).

    Метаданные проекта и выходные артефакты

    В typedoc.json обновите сведения, видимые в интерфейсе: 

    {
        "name": "@your-scope/your-library",
        "out": "docs",
        "readme": "README.md",
        "includeVersion": true,
        "cleanOutputDir": true,
        "hideGenerator": true,
        "titleLink": "https://github.com/<org>/<repo>",
    }
    • name — имя пакета в шапке.
    • titleLink — переход по клику на заголовок (обычно на репозиторий).

    Входные точки и интеграция с TypeScript

    Проверьте настройки входа и связку с TS:

    {
        "entryPoints": ["src/index.ts"],
        "entryPointStrategy": "resolve",
        "tsconfig": "tsconfig.build.json",
        "alwaysCreateEntryPointModule": true,
        "useTsLinkResolution": true,
    }
    • entryPoints/entryPointStrategy — документируется только то, что реэкспортировано из src/index.ts.
    • alwaysCreateEntryPointModule — стабилизирует структуру сайта, особенно при наличии внешних документов.
    • useTsLinkResolution — корректное разрешение {@link ...} в комментариях по правилам TS.

    Ссылки на исходники

    Включите кликабельные переходы к строкам исходного кода: 

    {
        "gitRevision": "HEAD",
        "sourceLinkTemplate": "https://github.com/<org>/<repo>/blob/{gitRevision}/{path}#L{line}",
        "sourceLinkExternal": true,
        "displayBasePath": "src",
    }
    • gitRevision — используйте HEAD или релизный тег (рекомендуется в CI).
    • displayBasePath — сокращает отображаемые пути в UI до относительных от src.
    • sourceLinkExternal — помечает ссылки как внешние.

    Отбор содержимого и фильтрация

    Поддерживайте в документации только публичные, задокументированные элементы: 

    {
        "excludeExternals": true,
        "excludePrivate": true,
        "excludeProtected": false,
        "excludeInternal": true,
        "excludeNotDocumented": true,
        "exclude": [
            "**/*.spec.ts",
            "**/*.test.ts",
            "**/__tests__/**",
            "**/__mocks__/**",
            "**/fixtures/**",
            "**/examples/**",
            "**/scripts/**",
        ],
    }
    • При необходимости временно ослабьте политику, установив "excludeNotDocumented": false.

    Организация навигации и порядок отображения

    Фиксируйте логическую структуру разделов и стабильную сортировку: 

    {
        "categorizeByGroup": true,
        "defaultCategory": "Misc",
        "categoryOrder": [
            "Core",
            "Builders",
            "Utils",
            "Types",
            "Experimental",
            "Misc",
        ],
        "sort": ["documents-first", "kind", "alphabetical", "source-order"],
        "navigationLinks": {
            "GitHub": "https://github.com/<org>/<repo>",
        },
    }
    • documents-first — страницы-мануалы отображаются выше API.
    • navigationLinks — ссылки в верхнем меню.

    Тема, плагины и видимость в интерфейсе

    Оформление и стартовые фильтры UI:

    {
        "theme": "typedoc-github-theme",
        "plugin": [
            "typedoc-github-theme",
            "typedoc-plugin-mdn-links",
            "typedoc-plugin-rename-defaults",
        ],
        "visibilityFilters": {
            "protected": false,
            "private": false,
            "inherited": true,
            "external": false,
        },
    }

    Внешние руководства и поиск

    Подключите собственные руководства (Markdown) и индексацию: 

    {
        "projectDocuments": ["manuals/*.md"],
        "searchInDocuments": true,
        "searchInComments": true,
    }

    Рекомендации для файлов в manuals/:

    • Добавляйте фронтматтер для заголовка и группировки: 

      ---
      title: Подробная настройка TypeDoc
      group: Руководства
      category: Guides
      ---

    • После первой сборки откройте docs/ и уточните фактический URL страницы. Если требуется, синхронизируйте его с navigationLinks.Guides.

    Подсветка синтаксиса

    Включите необходимые языки и темы подсветки:

    {
        "highlightLanguages": [
            "md",
            "markdown",
            "ts",
            "js",
            "json",
            "jsonc",
            "bash",
            "sh",
            "yaml",
            "yml",
            "text",
        ],
        "lightHighlightTheme": "github-light",
        "darkHighlightTheme": "github-dark",
    }
    • Для Markdown используйте обозначения md или markdown.
    • При необходимости добавляйте другие алиасы (например, toml, ini).

    Валидация и поведение сборки

    Настройки контроля качества:

    {
        "validation": {
            "notExported": true,
            "invalidLink": true,
            "notDocumented": true,
        },
        "treatWarningsAsErrors": false,
    }
    • В CI можно усилить политику, установив "treatWarningsAsErrors": true и фиксированный gitRevision на релизный тег.

    Сборка и локальная проверка

    Добавьте (или используйте готовые) скрипты:

    {
        "scripts": {
            "docs:build": "typedoc --options typedoc.json",
            "docs:serve": "npx http-server docs -o -c-1",
        },
    }
    • Сборка: npm run docs:build
    • Просмотр: npm run docs:serve

    Проверьте:

    • корректность ссылок на исходники,
    • наличие раздела с руководствами,
    • работу поиска по комментариям и документам,
    • подсветку блоков md/markdown.

    Контрольный список изменений (короткая версия)

    1. Обновить name, titleLink, navigationLinks.
    2. Задать gitRevision, sourceLinkTemplate, sourceLinkExternal, displayBasePath.
    3. Проверить вход API и TS-связку: entryPoints, entryPointStrategy, tsconfig, alwaysCreateEntryPointModule, useTsLinkResolution.
    4. Уточнить политику фильтрации: exclude* и список exclude.
    5. Зафиксировать структуру навигации: categorizeByGroup, categoryOrder, sort: ["documents-first", ...].
    6. Подключить мануалы: projectDocuments, фронтматтер, searchInDocuments/searchInComments.
    7. Настроить подсветку: highlightLanguages, lightHighlightTheme, darkHighlightTheme.
    8. Включить валидацию: validation, при необходимости ужесточить treatWarningsAsErrors.
    9. Собрать и проверить готовый сайт документации.
